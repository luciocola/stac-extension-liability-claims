== Conventions

=== Identifiers

The normative provisions in this specification are denoted by the URI `http://www.opengis.net/spec/liability-claims/1.1`

All requirements and conformance tests that appear in this document are denoted by partial URIs which are relative to this base.

=== Use of IETF RFC 2119

The key words "MUST", "MUST NOT", "REQUIRED", "SHALL", "SHALL NOT", "SHOULD", "SHOULD NOT", "RECOMMENDED", "MAY", and "OPTIONAL" in this document are to be interpreted as described in <<rfc2119>>.

=== UML Notation

This specification uses the Unified Modeling Language (UML) static structure diagrams as specified in Sub-clause 5.2 of <<iso19103>>. The UML diagrams in this specification follow ISO 19103 conventions:

* Classes are shown as rectangles with compartments for name, attributes, and operations
* Associations are shown as lines connecting classes
* Multiplicity is shown using the notation [min..max]
* Inheritance is shown with a hollow triangle pointing to the parent class
* Package dependencies are shown with dashed arrows
